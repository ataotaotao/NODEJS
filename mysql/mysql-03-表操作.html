修改数据库表

添加列名
xxxxxxxxxx
alter table 表名
    add 新列名 数据类型
    [约束条件][first(表示添加到最前面)][after 已存在的列名(表示添加列名后面);
修改列
xxxxxxxxxx
alter table 表名
    change 旧列名 新列名 新数据类型;
修改列的数据类型
xxxxxxxxxx
alter table 表名 modify 列名 数据类型
修改列的排列位置
xxxxxxxxxx
alter table 表名
    modify 列1 数据类型 first / after 列2;
删除列
xxxxxxxxxx
alter table 表名
    drop 列名;
修改表名
xxxxxxxxxx
alter table 表名 rename to 新表名;

 

删除数据库表

xxxxxxxxxx
drop table [if exists] 表1 , 表2 ,..... 表n；

如果不加 if exists ， 那么删除表的时候，就会报错，但是该删除的表还是会删除的。加了这个就会隐藏错误信息。

表分区

当用到的数据量很大时，我们就得想想如何去优化他，表分区就是一种

show plugins 可以查看是否支持表分区

数据库的表分区有两种方式，水平和垂直

  • 水平分区是将表的数据按行分割成不同的数据文件
  • 垂直分区则是将表的数据按列分割成不同的数据文件。

创建表分区

分区举例
xxxxxxxxxx
//先创建一个数据表
create table bookinfo(
    book_id int,
    book_name varchar(20)
);
//定义分区
partition by range(book_id)(
    partition p1 values less than (20109999),
    partition p2 values less than (20159999),
    partition p3 values less than MAXVALUE
);
/*
    通过对列进行划分，例如上面的是对book_id进行范围划分，分成了三部分，第一部分是小于2010年的，第二部分是2010-2015 ，第三部分是剩下的。
*/

如果进行了分区，那么可以进行分区查看

select * from reader partition(p2);
